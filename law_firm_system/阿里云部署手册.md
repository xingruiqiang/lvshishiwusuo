# 小型律所管理系统 - 阿里云部署手册

> 适用于：阿里云轻量应用服务器（2GB 内存）  
> 操作系统：Alibaba Cloud Linux 3 / CentOS 7+ / Ubuntu 20.04+

---

## 一、服务器准备

### 1.1 购买与登录

1. 登录阿里云控制台，购买 **轻量应用服务器**（2核 2GB 内存）。
2. 选择操作系统镜像：**Alibaba Cloud Linux 3** 或 **CentOS 7.9**。
3. 在控制台重置 root 密码，记录 **公网 IP**。
4. 本地通过 SSH 登录：
   ```bash
   ssh root@你的服务器公网IP
   ```

### 1.2 阿里云安全组 / 防火墙放通

在阿里云控制台 → 防火墙 / 安全组，添加以下规则：

| 端口 | 用途 | 授权对象 |
|------|------|----------|
| 22 | SSH 远程登录 | 你的办公 IP |
| 5066 | 律所系统 | 0.0.0.0/0 或你的办公 IP |

服务器内防火墙放行（可选）：
```bash
firewall-cmd --permanent --add-port=5066/tcp
firewall-cmd --reload
```

## 三、生产环境部署

### 3.1 创建必要目录

```bash
cd /opt/law_firm_system
mkdir -p backup upload static/uploads logs
chmod -R 777 upload static/uploads
```

### 3.2 使用 Gunicorn 启动

2G 内存服务器建议 **1 个 worker**：

```bash
cd /opt/law_firm_system
gunicorn -w 1 -b 0.0.0.0:5066 --timeout 120 --access-logfile logs/access.log --error-logfile logs/error.log app:app
```

### 3.3 配置 Systemd 开机自启

```bash
# 查找 gunicorn 路径
which gunicorn
# 通常输出：/usr/local/bin/gunicorn
```

创建服务文件：
```bash
cat > /etc/systemd/system/lawfirm.service << 'EOF'
[Unit]
Description=Law Firm Management System
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/law_firm_system
ExecStart=/usr/local/bin/gunicorn -w 1 -b 0.0.0.0:5066 --timeout 120 --access-logfile /opt/law_firm_system/logs/access.log --error-logfile /opt/law_firm_system/logs/error.log app:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

启动并设为开机自启：
```bash
systemctl daemon-reload
systemctl start lawfirm
systemctl enable lawfirm
systemctl status lawfirm
```

常用命令：
```bash
systemctl start lawfirm      # 启动
systemctl stop lawfirm       # 停止
systemctl restart lawfirm    # 重启
systemctl status lawfirm     # 查看状态
journalctl -u lawfirm -f     # 查看实时日志
```

## 五、自动备份（每天 20:00）

### 5.1 创建备份脚本

```bash
cat > /opt/law_firm_system/backup.sh << 'EOF'
#!/bin/bash
# 律所系统数据库自动备份脚本

PROJECT_DIR="/opt/law_firm_system"
DB_FILE="$PROJECT_DIR/db/law_db.sqlite"
BACKUP_DIR="$PROJECT_DIR/backup"
DATE=$(date +%Y%m%d_%H%M%S)

# 备份数据库
cp "$DB_FILE" "$BACKUP_DIR/law_db_${DATE}.sqlite"

# 删除 30 天前的备份
find "$BACKUP_DIR" -name "law_db_*.sqlite" -mtime +30 -delete

# 记录备份日志
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 数据库备份完成: law_db_${DATE}.sqlite" >> "$PROJECT_DIR/logs/backup.log"
EOF

chmod +x /opt/law_firm_system/backup.sh
```

### 5.2 配置定时任务（每天 20:00 执行）

```bash
crontab -e
```

添加以下内容：
```
0 20 * * * /opt/law_firm_system/backup.sh
```

验证定时任务：
```bash
crontab -l
```

### 5.3 手动执行备份

```bash
/opt/law_firm_system/backup.sh
```

### 5.4 下载备份到本地

```bash
# 在本地 Windows 执行
scp root@你的服务器公网IP:/opt/law_firm_system/backup/law_db_*.sqlite C:\Users\lvping\Desktop\
```

## 六、2G 内存优化

### 6.1 配置 Swap

```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

### 6.2 日志管理

```bash
yum install -y logrotate   # CentOS
apt install -y logrotate   # Ubuntu

cat > /etc/logrotate.d/lawfirm << 'EOF'
/opt/law_firm_system/logs/*.log {
    daily
    missingok
    rotate 7
    compress
    delaycompress
    notifempty
    create 0644 root root
}
EOF
```

## 七、故障排查

### 7.1 服务无法启动

```bash
journalctl -u lawfirm -n 50          # 查看服务日志
cat /opt/law_firm_system/logs/error.log  # 查看应用日志
netstat -tunlp | grep 5066           # 检查端口占用
```

### 7.2 数据库恢复

```bash
cd /opt/law_firm_system
systemctl stop lawfirm
cp backup/law_db_xxxx.sqlite db/law_db.sqlite   # 用备份恢复
systemctl start lawfirm
```

### 7.3 内存不足

```bash
free -h                              # 查看内存
dmesg | grep -i 'out of memory'      # 查看 OOM 日志
```

### 7.4 上传失败

```bash
ls -ld /opt/law_firm_system/upload
chmod -R 777 /opt/law_firm_system/upload
chmod -R 777 /opt/law_firm_system/static/uploads
```

## 九、安全建议

1. **修改默认密码**：首次登录后立即修改 admin 密码
2. **限制访问来源**：阿里云安全组中 5066 端口只放行办公室公网 IP
3. **定期备份**：数据库已配置每天 20:00 自动备份
4. **关闭调试模式**：生产环境 `debug=False`（已在 config.py 中配置）
5. **定期更新系统**：`yum update` 或 `apt update && apt upgrade`
