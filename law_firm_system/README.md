# 小型律所管理系统 V1.0

## 项目简介

小型律所管理系统是一套专为中小型律师事务所设计的轻量化管理系统，采用 Flask + SQLite 架构，适合在 2GB 内存的阿里云 Linux 轻量服务器上部署运行。

### 系统特点

- **轻量化架构**：采用 Flask 框架 + SQLite 数据库，资源占用低，适合小型服务器部署
- **功能全面**：涵盖客户管理、案件管理、文书模板、案例库、财务管理等核心业务
- **易于部署**：提供详细的部署文档，支持一键初始化和快速启动
- **权限管理**：支持管理员、律师、助理三种角色，权限分级管理
- **数据安全**：支持数据备份恢复，保障律所数据安全

## 功能模块

### 1. 首页工作台
- 统计卡片：在办案件数、本月营收、开庭案件、未开票收款
- 待办提醒：开庭提醒、上诉期提醒、举证期提醒

### 2. 客户管理
- 客户信息录入、编辑、查询
- 支持标签分类（个人/企业/顾问单位）
- 客户详情页展示关联案件
- Excel 导入导出功能

### 3. 案件管理
- 案件信息采集（案号、类型、 court、承办律师等）
- 案件进度跟踪（立案→举证→开庭→判决→执行→归档）
- 开庭日期提醒，自动生成待办事项
- 案件附件上传管理

### 4. 法律文书库
- 文书模板分类管理（诉讼文书、非诉文书、合同协议等）
- 支持 Word 文档导出
- 系统预置10个常用文书模板
- 支持自定义模板编辑

### 5. 案例资料库
- 典型案例录入和检索
- 支持关键词、法条、裁判要点搜索
- 案例收藏功能
- 支持5个示例案例

### 6. 财务管理
- 收费管理：记录律师费、代理费等收入
- 支出管理：记录办公费、案件成本等支出
- 财务报表：按月统计收支情况
- 发票管理：跟踪开票状态

### 7. 系统管理
- 用户管理：添加/编辑用户，分配角色权限
- 律所信息配置：名称、信用代码、银行账户等
- 数据备份：一键备份数据库

## 技术架构

### 后端
- **框架**：Flask 2.3.3
- **数据库**：SQLite 3（通过 SQLAlchemy ORM）
- **文档处理**：python-docx（Word 导出）

### 前端
- **模板引擎**：Jinja2
- **CSS 框架**：Bootstrap 5
- **图标**：Bootstrap Icons

### 部署环境
- **操作系统**：Linux（CentOS 7+ / Ubuntu 18.04+）
- **Python 版本**：3.9+
- **内存要求**：2GB 及以上
- **Web 服务器**：Gunicorn（生产环境）

## 使用说明

### 1. 客户管理

**新增客户**
1. 进入"客户管理" → "客户列表"
2. 点击"新增客户"按钮
3. 填写客户信息（姓名/企业名称、身份证号、电话、地址等）
4. 选择客户标签（个人/企业/顾问单位）
5. 点击"保存"完成

**客户导入**
1. 在客户列表页面点击"导入"按钮
2. 下载导入模板 Excel 文件
3. 按照模板格式填写客户信息
4. 上传 Excel 文件完成导入

### 2. 案件管理

**新增案件**
1. 进入"案件管理" → "案件列表"
2. 点击"新增案件"按钮
3. 选择客户、填写案号、案件类型、受理法院等
4. 选择承办律师、输入对方当事人
5. 设置收案日期、开庭日期、审理阶段
6. 点击"保存"完成

**案件进度更新**
1. 在案件列表中找到对应案件
2. 点击"编辑"按钮
3. 修改"审理阶段"字段（立案→举证→开庭→一审判决→二审→执行→归档）
4. 点击"保存"完成

### 3. 文书模板库

**使用文书模板**
1. 进入"文书模板库" → "模板列表"
2. 找到需要的文书模板（如"民事起诉状"）
3. 点击"预览"查看模板内容
4. 点击"导出Word"下载 Word 文档
5. 在 Word 中替换变量（如[原告姓名]、[被告姓名]等）

**新增自定义模板**
1. 进入"文书模板库" → "模板列表"
2. 点击"新增模板"按钮
3. 填写模板标题、选择类型、输入模板内容
4. 在内容中使用变量占位符（如[当事人姓名]、[案由]等）
5. 点击"保存"完成

### 4. 案例资料库

**检索案例**
1. 进入"案例资料库" → "案例列表"
2. 输入关键词（如"合同纠纷"、"违约金"）
3. 选择案例类型（民事/刑事/劳动等）
4. 点击"搜索"查看结果

**收藏案例**
1. 在案例列表中找到有价值的案例
2. 点击"收藏"按钮
3. 收藏的案例会在"我的收藏"中显示

### 5. 财务管理

**记录收费**
1. 进入"财务管理" → "收费管理"
2. 点击"新增收费"按钮
3. 选择关联案件、填写收费金额、收费日期、收费方式
4. 设置发票状态（未开票/已开票）
5. 点击"保存"完成

**查看财务报表**
1. 进入"财务管理" → "财务报表"
2. 选择查询月份（如"2026-06"）
3. 查看本月收入、支出、净利润统计
4. 查看支出分类统计（办公、案件、其他）

### 6. 系统管理

**添加用户**
1. 以管理员身份登录
2. 进入"系统管理" → "用户管理"
3. 点击"新增用户"按钮
4. 填写用户名、密码、真实姓名、角色（管理员/律师/助理）
5. 点击"保存"完成

**配置律所信息**
1. 以管理员身份登录
2. 进入"系统管理" → "律所信息"
3. 填写律所名称、信用代码、法定代表人、成立日期、地址、电话、邮箱
4. 填写开户行、银行账号（用于发票和收费凭证）
5. 点击"保存"完成

**数据备份**
1. 以管理员身份登录
2. 进入"系统管理" → "数据备份"
3. 点击"立即备份"按钮
4. 系统会自动备份数据库到 backup/ 目录

## 常见问题

### 1. 登录失败怎么办？

- 检查用户名和密码是否正确
- 确认账号状态是否为"启用"（管理员可以在"用户管理"中启用账号）
- 如果忘记密码，需要管理员重置密码

### 2. 无法访问系统怎么办？

- 检查应用是否启动：```ps aux | grep python```
- 检查端口是否被占用：```netstat -tunlp | grep 5066```
- 检查防火墙是否开放5066端口

### 3. 上传文件失败怎么办？

- 检查 upload/ 目录是否存在且有写入权限
- Linux 系统执行：```mkdir -p upload && chmod 777 upload```

### 4. 导出 Word 失败怎么办？

- 检查是否安装了 python-docx：```pip list | grep python-docx```
- 如果没有安装，执行：```pip install python-docx```

### 5. 数据库损坏怎么办？

- 使用备份文件恢复：将 backup/ 目录中的备份文件复制到 db/ 目录
- 重命名为 law_db.sqlite
- 重启应用

## 已知问题

### 1. 系统设置 - 新增用户报错
- **问题描述**：在系统管理 → 用户管理中点击"新增用户"按钮时报错
- **原因**：缺少 `/system/user/add` 路由和对应的 HTML 模板
- **状态**：待修复
- **修复优先级**：高

### 2. 客户管理 - 导出Excel报错
- **问题描述**：在客户列表页面点击"导出"按钮时报错："The requested URL was not found on the server"
- **原因**：缺少 `/client/export` 路由（Excel导出功能）
- **状态**：待修复
- **修复优先级**：中

### 3. 文书模板库 - 预览按钮报错
- **问题描述**：在文书模板列表页面点击"预览"按钮时报错："Not Found"
- **原因**：缺少 `/document/preview/<id>` 路由和对应的 HTML 模板
- **状态**：待修复
- **修复优先级**：高
